

# Landlord Verification, Subscription & Profile System

## Overview

This plan adds a complete landlord verification and subscription system with:
- A **landlord_profiles** table for storing verification details (ID number, KRA PIN, documents, phone)
- A **subscriptions** table to track paid subscription status
- A verification workflow where admins review landlord details and documents before approving
- A **free tier limit of 5 properties**; beyond that, landlords must subscribe at KES 2,000/month via M-Pesa or Stripe
- A **Landlord Profile page** where landlords can view/edit personal and verification details
- A billing page where landlords can view subscription status and pay for subscription
- An enhanced **Admin Landlords page** that shows real landlord data (not mocked) including documents, phone, verification status, and subscription status.
- In admin, add a revenue page that shows revenue generated by subscriptions and Airbnb listings 
- Admin dashboard should show real statistics from the app

## Database Changes

### New Table: `landlord_profiles`

Stores extended landlord verification data separate from the general `profiles` table.

| Column | Type | Notes |
|--------|------|-------|
| id | uuid (PK) | Auto-generated |
| user_id | uuid (unique, not null) | References the landlord's auth user |
| id_number | text | National ID number |
| kra_pin | text | KRA PIN |
| business_phone | text | Phone number for clients |
| documents | text[] | Array of storage URLs (PDF/image uploads) |
| verification_status | text | `unverified`, `pending`, `verified`, `rejected` (default: `unverified`) |
| verification_notes | text | Admin notes on verification decision |
| verified_at | timestamptz | When admin verified |
| verified_by | uuid | Admin user who verified |
| created_at / updated_at | timestamptz | Timestamps |

**RLS Policies:**
- Landlords can SELECT/INSERT/UPDATE their own row (by `user_id = auth.uid()`)
- Admins can SELECT and UPDATE all rows (for verification workflow)

### New Table: `subscriptions`

Tracks active subscriptions for landlords.

| Column | Type | Notes |
|--------|------|-------|
| id | uuid (PK) | Auto-generated |
| user_id | uuid (unique, not null) | The landlord |
| status | text | `active`, `expired`, `cancelled` (default: `expired`) |
| plan | text | e.g., `basic` (default: `basic`) |
| amount | numeric | 2000 |
| payment_method | text | `mpesa` or `stripe` |
| payment_reference | text | Transaction reference |
| starts_at | timestamptz | Subscription start |
| expires_at | timestamptz | When it expires |
| created_at / updated_at | timestamptz | Timestamps |

**RLS Policies:**
- Landlords can SELECT their own subscription
- Admins can SELECT and UPDATE all subscriptions

### New Storage Bucket: `landlord-documents`

A public bucket for storing landlord verification documents (ID copies, KRA certificates, etc.).

**Storage RLS:**
- Landlords can upload to their own folder (`user_id/...`)
- Admins can read all files

## New Pages & Components

### 1. Landlord Profile Page (`src/pages/landlord/LandlordProfilePage.tsx`)

A dedicated page in the landlord portal where landlords can:
- View and edit personal details (full name, email, phone, avatar) -- similar to the user profile page
- Fill in verification details: ID Number, KRA PIN, business phone
- Upload verification documents (PDF or image, up to 5 files)
- See their verification status (Unverified / Pending Review / Verified / Rejected with notes)
- See their subscription status (Free tier / Active subscription / Expired)
- Subscribe or renew if they have 5+ properties

**Layout:**
- Top section: Profile card with avatar, name, email (editable)
- Middle section: "Verification Details" card with form fields for ID number, KRA PIN, business phone, and document upload area
- Bottom section: "Subscription" card showing current plan, expiry, and a subscribe/renew button

**Responsive design:**
- Single column on mobile, two-column layout on desktop where profile and verification sit side by side, subscription below

### 2. Subscription Payment Dialog (`src/components/landlord/SubscriptionPaymentDialog.tsx`)

A dialog that appears when a landlord needs to subscribe. It:
- Shows the plan details (KES 2,000/month)
- Lets the user choose M-Pesa or Stripe
- For M-Pesa: prompts for phone number, calls the existing M-Pesa STK push edge function (adapted for subscriptions)
- For Stripe: calls the existing Stripe checkout edge function (adapted for subscriptions)
- On success, creates/updates the subscription record

### 3. Document Upload Component (`src/components/landlord/DocumentUpload.tsx`)

A reusable component for uploading verification documents:
- Accepts PDF and image files (jpg, png, pdf)
- Max 5 documents
- Uploads to the `landlord-documents` storage bucket
- Shows thumbnails for images and file icons for PDFs
- Allows deletion of uploaded documents

## Edge Function Changes

### 8. New Edge Function: `process-subscription` (`supabase/functions/process-subscription/index.ts`)

Handles subscription payment processing:
- Authenticates the landlord
- Validates they have landlord role
- Checks if M-Pesa or Stripe payment method
- Creates/updates the subscription record with `active` status and `expires_at` = now + 30 days
- Returns success with subscription details

### 9. Update `supabase/config.toml`

Add the new edge function entry:
```
[functions.process-subscription]
verify_jwt = false
```

## Business Logic Summary

```text
Landlord Registration Flow:
1. User signs up as landlord
2. Landlord fills in profile + verification details (ID, KRA, docs)
3. Submits for verification -> status becomes "pending"
4. Admin reviews and approves/rejects
5. If approved -> status becomes "verified", landlord can list properties

Property Listing Flow:
1. Landlord clicks "Add Property"
2. System checks: Is landlord verified? 
   - No -> Redirect to profile page with message
3. System checks: Property count >= 5 AND no active subscription?
   - Yes -> Show subscription payment dialog
   - No -> Allow listing
4. Property is created with "pending" status for admin approval

Subscription Flow:
1. Landlord hits 5-property limit
2. Subscription dialog appears
3. Landlord chooses M-Pesa or Stripe
4. Payment processed via edge function
5. Subscription record created (30-day validity)
6. Landlord can now list unlimited properties
```
